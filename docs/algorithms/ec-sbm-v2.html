EC-SBM v2

four-stage constructive + residual SBM + block-preserving rewire + hybrid match

v2 runs a four-stage pipeline on a 20-node input: construct k-edge-connected cluster cores (stage 2), sample a residual SBM over all blocks with block-preserving 2-opt rewire (stage 3), then top up missing degree with the hybrid matcher (stage 4). Unlike v1, stage 2 places only the constructive edges; the SBM call is deferred to stage 3a where the accounting is cleaner.

Default settings shown here: profile outlier_mode=excluded, gen_outlier_mode=combined, --edge-correction rewire, --algorithm hybrid. note the registry default in generators/ec-sbm-v2.sh is true_greedy, but hybrid is usually the more robust choice, so this viz walks through the hybrid path.
